# ZK-Compressed Equipment & Authority Management System

A high-performance, modular Solana program built with the **Anchor Framework** and **Light Protocol (ZK Compression)**. This system provides a comprehensive solution for managing administrative permissions, feature configurations, and scalable asset minting (Artifact Boxes).

## 🏗 System Architecture

The program follows a layered design to ensure security, flexibility, and cost-efficiency:

1.  **Authority Layer (`AuthorityRegistry`)**:
    *   Decentralized permission management based on `feature_id`.
    *   Supports a **Two-step Transfer** mechanism (Propose & Accept) to prevent accidental ownership loss.
2.  **Metadata Layer (`IpfsConfig`)**:
    *   Links off-chain IPFS URIs to specific features.
    *   Uses `realloc` for dynamic storage sizing.
3.  **Operational Layer (`EquipmentConfig`)**:
    *   Provides global toggles for business logic (e.g., `pause` and `mint_enabled`).
    *   Acts as a circuit breaker for emergency management.
4.  **Asset Layer (`ArtifactBox`)**:
    *   Utilizes **ZK-Compression** to store equipment random_seed data in compressed state trees.
    *   Drastically reduces storage (Rent) costs compared to traditional Solana accounts.

---

## 🚀 Core Features

-   **Modular Governance**: Use unique `feature_id` strings to isolate permissions for different game modules or services.
-   **Secure Admin Transfer**: Admin changes require the new administrator to sign an "Accept" transaction, ensuring the address is valid and active.
-   **ZK-Compressed Assets**: Leverages Light Protocol for minting "Artifact Boxes" which exist as compressed states, allowing for millions of items at minimal cost.
-   **Operational Control**: Administrators can instantly pause specific features or disable minting through the `EquipmentConfig`.

## 🛠 Instruction Reference


The program instructions are categorized into three functional modules:

### 1. Authority Registry (Governance)
Manages administrative rights for specific features using a secure two-step transfer flow.

| Instruction | Parameters | Signer | Description |
| :--- | :--- | :--- | :--- |
| `authority_registry_initialize` | `feature_id` | Initial Admin | Initialize authority for a feature. |
| `authority_registry_propose` | `feature_id` | Current Admin | Propose a new admin for the feature. |
| `authority_registry_accept` | `feature_id` | New Admin | New admin accepts and finalized the transfer. |
| `authority_registry_cancel` | `feature_id` | Admin/Pending | Cancel an active transfer proposal. |

### 2. Configuration & Operational Controls
Handles metadata and business logic toggles (circuit breakers).

| Instruction | Parameters | Signer | Description |
| :--- | :--- | :--- | :--- |
| `ipfs_config_initialize` | `feature_id`, `config_uri` | Feature Admin | Setup IPFS metadata URI. |
| `ipfs_config_update` | `feature_id`, `config_uri` | Feature Admin | Update IPFS URI (supports realloc). |
| `equipment_config_initialize` | `feature_id` | Feature Admin | Initialize pause/mint toggles. |
| `equipment_config_update` | `feature_id`, `pause?`, `mint_enabled?` | Feature Admin | Update operational status. |

### 3. ZK Asset Minting (Light Protocol)
Handles the creation of compressed assets using zero-knowledge proofs.

| Instruction | Parameters | Description |
| :--- | :--- | :--- |
| `generate_artifact_box` | `proof_bytes`, `address_tree_info_bytes`, `output_state_tree_index`, `artifact_id`, `commitment` | Mints a ZK-compressed Artifact Box. Internally deserializes proofs and tree info to interface with Light Protocol. |

## 🔑 Key Parameters Note
For `generate_artifact_box`, the `proof_bytes` and `address_tree_info_bytes` must be serialized using Borsh before being passed to the instruction. The program will validate these against the Light Protocol state tree.
